# Cómo Usar el Formulario de Flujo de WhatsApp en E-SMART360: Guía 2025

En la era digital de 2025, WhatsApp es mucho más que una aplicación de mensajería. La función **WhatsApp Flow** permite a las empresas crear formularios interactivos y estructurados directamente dentro de la interfaz de chat. Estos formularios recopilan información de clientes, automatizan procesos y aumentan el compromiso. E-SMART360, como plataforma de automatización líder, ahora es compatible con WhatsApp Flow para negocios de todos los tamaños.


> **¿Qué encontrarás en esta guía?** Aprenderás desde los fundamentos de WhatsApp Flow hasta la configuración paso a paso de formularios interactivos, integración con APIs HTTP externas, casos de éxito reales, mejores prácticas y automatización de conversaciones complejas con flujo de entrada de usuario en webchat.

## ¿Qué es WhatsApp Flow en 2025?

WhatsApp Flow es una funcionalidad de la plataforma WhatsApp Business que permite a las empresas construir formularios interactivos y flujos de trabajo dentro del propio chat. Estos formularios permiten a los clientes enviar detalles, tomar decisiones y completar acciones sin salir de WhatsApp. En 2025, esta herramienta ha evolucionado con nuevas capacidades que empresas de todos los tamaños necesitan.

### Novedades de WhatsApp Flow en 2025

- **Sugerencias impulsadas por IA**: La IA ahora recomienda diseños de formularios basados en las necesidades de tu negocio, analizando tu industria y el tipo de datos que necesitas recopilar.
- **Integraciones avanzadas**: La funcionalidad se comunica con sistemas CRM, plataformas de pago y rastreadores de inventario de forma nativa.
- **Analíticas en tiempo real**: Revisa el rendimiento de los formularios utilizando datos en vivo, incluyendo tasas de finalización, abandono y tiempo promedio de respuesta.
- **Compatibilidad multiplataforma**: Los flujos funcionan sin problemas en WhatsApp, webchat, Facebook Messenger e Instagram.
- **Campos personalizables**: Los datos recopilados pueden mapearse directamente a campos personalizados en el perfil del suscriptor para su posterior procesamiento y segmentación.

Estas mejoras convierten a WhatsApp Flow en una herramienta indispensable para los planes de comunicación empresarial moderna y automatización de procesos.

## ¿Cómo funciona WhatsApp Flow?

WhatsApp Flow en 2025 permite a las empresas crear formularios personalizados que guían a los clientes a través de pasos estructurados. Estos formularios pueden vincularse a chatbots o enviarse manualmente en chats en vivo. Así es como opera:


### Creación del formulario

Usando la interfaz de arrastrar y soltar de E-SMART360, las empresas añaden elementos como campos de texto, casillas de verificación, botones de opción y campos de fecha para construir formularios completos. La IA puede sugerir diseños basados en tu tipo de negocio y necesidades específicas.

### Integración con chatbot

Los formularios se vinculan a chatbots, activándose automáticamente según las entradas del cliente, como una palabra clave específica o una acción determinada dentro de la conversación.

### Recolección de datos

Los endpoints definidos pueden responder en tiempo real, y otros servicios pueden integrarse mediante APIs HTTP para procesar la información. Los datos se almacenan tanto en las analíticas de WhatsApp Flow como en campos personalizados del suscriptor.

### Respuestas del sistema

Después de que el formulario ha sido enviado, el sistema puede enviar mensajes automáticos de confirmación o marketing. También puede activar flujos de trabajo secundarios como envío de correos electrónicos o creación de tickets de soporte.

### Análisis y optimización

Los datos recopilados pueden analizarse en tiempo real para identificar tendencias, mejorar formularios y optimizar la experiencia del cliente en iteraciones sucesivas.


> **Dato clave:** La recolección de datos puede ser una tarea tediosa, especialmente con la creciente demanda de satisfacción del cliente. Con E-SMART360, este proceso se simplifica enormemente al automatizar toda la cadena de recolección, procesamiento y almacenamiento de datos.

## Ejemplos prácticos de negocio: WhatsApp Flow para 2025

WhatsApp Flow se puede utilizar para muchos objetivos diferentes, haciendo que su uso sea único para cada empresa. Aquí algunos ejemplos concretos organizados por industria:



### Salud

**Programación de citas médicas:** Los pacientes pueden chatear para ingresar sus datos médicos requeridos y seleccionar un médico que deseen visitar. El formulario recopila síntomas, preferencias de horario y seguro médico. El flujo puede verificar disponibilidad en tiempo real mediante integración API y confirmar la cita automáticamente.

**Flujo paso a paso:**
1. El paciente envía "Agendar cita médica"
2. El bot pregunta la especialidad requerida
3. El paciente selecciona de una lista desplegable
4. El bot muestra horarios disponibles en tiempo real
5. El paciente confirma fecha y hora
6. El bot envía confirmación con recordatorio 24 horas antes

### Restaurantes

**Reseñas de clientes:** Los restaurantes comparten formularios de calificación para que los comensales los llenen después de comer, ayudando a mejorar sus servicios. Pueden incluir campos de valoración por plato, servicio y ambiente.

**Flujo paso a paso:**
1. Después de la visita, el bot envía formulario de feedback
2. El cliente califica del 1 al 5 en comida, servicio y ambiente
3. Campo opcional para comentarios adicionales
4. Si la calificación es baja (< 3), se notifica al gerente automáticamente
5. Se envía un cupón de descuento como agradecimiento

### E-commerce

**Recomendaciones de productos:** Las tiendas en línea crean flujos orientados al cliente para solicitar información sobre preferencias y ofrecer servicios que coincidan con esos intereses.

**Flujo paso a paso:**
1. El cliente envía "Recomiéndame productos"
2. El bot pregunta la categoría de interés
3. El cliente selecciona una categoría
4. El bot pregunta el rango de precio
5. El cliente ingresa su presupuesto
6. El bot muestra 3 a 5 productos recomendados con imágenes y precios
7. El cliente puede comprar directamente desde el chat

### Soporte técnico

**Solicitudes de soporte:** Los formularios permiten a las empresas recibir detalles de los problemas de los clientes para mejorar la efectividad del soporte en cuanto a velocidad de resolución.

**Flujo paso a paso:**
1. El cliente envía "Ayuda" o "Soporte técnico"
2. El bot pregunta la categoría del problema
3. El cliente selecciona: facturación, técnico o ventas
4. El bot solicita una descripción detallada del problema
5. El cliente ingresa la descripción
6. El bot crea un ticket automáticamente en el sistema de soporte
7. El bot confirma el número de ticket y el tiempo estimado de respuesta

### Más aplicaciones por industria

- **Educación**: Inscripción a cursos, solicitud de materiales educativos, programación de tutorías personalizadas.
- **Bienes raíces**: Agendar visitas a propiedades, filtrar preferencias de búsqueda, solicitar cotizaciones personalizadas.
- **Servicios financieros**: Solicitud de préstamos, actualización de datos bancarios, consulta de saldos y movimientos.
- **Hotelería**: Check-in anticipado, solicitud de servicios a la habitación, reserva de restaurante del hotel.
- **Logística**: Seguimiento de envíos en tiempo real, programación de entregas, confirmación de recepción de paquetes.

## Cómo crear un formulario de flujo de WhatsApp en E-SMART360

Completar un formulario de flujo de WhatsApp dentro de E-SMART360 es bastante sencillo. Sigue esta guía paso a paso:

### Paso 1: Accede al panel de E-SMART360

Regístrate o inicia sesión en tu cuenta de E-SMART360 desde el panel de control. Una vez dentro, verás el menú principal con todas las opciones disponibles. Asegúrate de tener una cuenta de WhatsApp Business ya configurada y vinculada a la plataforma.


> **Requisito previo:** Antes de crear formularios de flujo, asegúrate de que tu número de teléfono esté registrado en WhatsApp Cloud API y verificado. También necesitas al menos un bot de respuestas configurado en el Bot Manager.

### Paso 2: Selecciona el bot de WhatsApp

En el panel de control, navega hasta la sección de bots y elige tu cuenta de negocio de WhatsApp vinculada. Si tienes múltiples números, selecciona el que utilizarás para el formulario de flujo.

### Paso 3: Inicia un nuevo flujo

Dentro del Bot Manager, busca la sección **WhatsApp Flow Forms** o **Formularios de Flujo de WhatsApp**. Haz clic en el botón **Crear** para abrir el constructor de formularios.

### Paso 4: Construye el formulario con el editor visual

El constructor de formularios te presenta un lienzo vacío donde puedes arrastrar y soltar componentes. Los elementos disponibles incluyen:

- **Campos de texto**: Para entradas cortas como nombre, email o teléfono.
- **Área de texto**: Para respuestas largas como comentarios o descripciones detalladas.
- **Casillas de verificación**: Para selección múltiple de opciones como intereses o preferencias.
- **Botones de opción**: Para seleccionar una sola opción de una lista.
- **Campo de fecha**: Para recopilar fechas como cumpleaños o fechas de cita.
- **Encabezado**: Para añadir títulos y separar secciones del formulario.
- **Botón de envío**: Para finalizar y procesar el formulario completado.

La IA puede sugerir diseños basados en tus requerimientos y tipo de negocio, optimizando la disposición de los campos para máxima efectividad.

### Paso 5: Configura los detalles del flujo

Cada formulario requiere una configuración específica que debes completar cuidadosamente. A continuación se detalla cada campo:

**Nombre del flujo (Flow Name):**
Debes asignar un nombre único a tu flujo. Cada vez que crees un nuevo formulario de flujo, ten en cuenta que todos los nombres deben ser únicos. Si intentas usar un nombre existente, el sistema mostrará un error indicando que el nombre ya está en uso.

**Categoría del flujo (Flow Categories):**
Selecciona qué tipo de formulario deseas crear desde esta categoría. Haz clic en esta opción y aparecerá un menú desplegable con muchas opciones predefinidas que se adaptan a diferentes casos de uso, como registros, encuestas, solicitudes de soporte y más.

**Nombre único de pantalla (Screen Unique Name):**
Aquí asigna un nombre único sin espacios. Ten cuidado al escribir, porque mantener espacios entre palabras causará errores de validación. Usa guiones bajos o camelCase para separar palabras. Por ejemplo: `formulario_registro_cliente` o `formularioRegistroCliente`.

**Título del formulario (Form Title):**
Asigna un título descriptivo para tu flujo de WhatsApp. Este título será visible para los usuarios como encabezado principal del formulario. Debe ser claro, descriptivo y atractivo. Por ejemplo: "Registro para consulta médica" o "Solicitud de cotización personalizada".

### Paso 6: Selecciona una API HTTP

La opción **Seleccionar API HTTP** te permite elegir una conexión API HTTP previamente configurada en la plataforma. Al hacer clic se abrirá un menú desplegable con todas tus conexiones API HTTP ya creadas. Solo selecciona una y estarás listo.


### ¿Necesitas ayuda para configurar la API HTTP?

Si no has configurado ninguna API HTTP aún, puedes hacerlo desde **Integración > API HTTP** en el panel principal de E-SMART360. Haz clic en **Crear** y proporciona los siguientes detalles:
- Nombre de la API (ej. "Crear Usuario WordPress")
- Método HTTP (GET, POST, PUT, DELETE)
- URL del endpoint (ej. https://tudominio.com/api/v1/usuarios)
- Encabezados HTTP (Content-Type, Authorization, etc.)
- Cuerpo de la solicitud con campos estáticos o dinámicos
- Verifica la conexión con el botón "Verificar Conexión"
- Guarda la API y estará disponible en el menú desplegable del formulario de flujo

Para más detalles, consulta la sección Integración de API HTTP más adelante en esta guía.

### Paso 7: Configura la respuesta posterior al envío

Esta opción te permite definir una respuesta automática que se enviará después de que tu cliente envíe el formulario. Puedes seleccionar una respuesta existente del menú desplegable o crear una nueva haciendo clic en el botón **Añadir**.


> **Ejemplos de respuestas de confirmación efectivas:**
- "¡Gracias por completar el formulario! Te contactaremos en las próximas 24 horas hábiles."
- "Formulario recibido exitosamente. Tu número de confirmación es #12345. Guarda este número para referencia futura."
- "¡Gracias por tu feedback! Hemos registrado tus comentarios y nuestro equipo los revisará pronto para mejorar nuestro servicio."

### Paso 8: Guarda datos en campos personalizados

Aunque puedes revisar los datos de los clientes desde las analíticas de WhatsApp Flow, también puedes elegir un campo personalizado para cada elemento del formulario. Después de arrastrar un elemento y abrirlo, verás un menú desplegable llamado **Campo Personalizado** al final de las opciones de configuración.

Los campos personalizados se gestionan desde el Administrador de Suscriptores. Puedes crear tantos campos como necesites para almacenar diferentes tipos de datos. Las ventajas de usar campos personalizados incluyen:

- Los datos quedan vinculados al perfil del suscriptor para consultas futuras
- Puedes segmentar audiencias basándote en los datos recopilados
- Los campos personalizados pueden usarse en campañas de broadcast para personalizar mensajes
- Facilitan la integración con sistemas externos mediante APIs

### Paso 9: Revisión final

Antes de publicar, verifica si tu formulario tiene errores o correcciones pendientes. Es importante saber que no podrás editarlo después de publicarlo, por lo que debes revisar cuidadosamente:

- Ortografía y redacción de los campos y títulos
- Validaciones de campos (obligatorios vs. opcionales)
- Conexión con la API HTTP seleccionada
- Respuesta de confirmación configurada
- Mapeo de campos personalizados
- Orden y flujo lógico de los elementos del formulario

### Paso 10: Vista previa y lanzamiento

Usa la opción **Vista previa del flujo** o **Preview Flow** para ver cómo se muestra el formulario en WhatsApp. Esto te permite ver exactamente lo que verá el cliente antes de publicarlo. Verifica:

- La apariencia visual del formulario en diferentes tamaños de pantalla
- El flujo de navegación entre campos
- El comportamiento de los botones y enlaces
- Los mensajes de error y validación

Cuando todo esté listo, haz clic en el botón **Publicar**. Una vez publicado, el formulario estará disponible para ser vinculado a tu chatbot.

### Paso 11: Vincula al chatbot

Para usar el flujo, conéctalo a la ruta de conversación de tu chatbot en el constructor visual de flujos. Sigue estos pasos:

1. Ve al Bot Manager y haz clic en **Crear** para abrir el constructor visual de flujos
2. Configura el componente **Iniciar flujo del bot** (Start Bot Flow)
3. Conéctalo al componente **Flujo de WhatsApp** (WhatsApp Flow)
4. Guarda el flujo del bot
5. Activa el bot para ponerlo en funcionamiento


> **Importante:** Después de publicar el formulario y conectarlo al chatbot, prueba el flujo completo enviando un mensaje desde un número de prueba. Verifica que el formulario se muestre correctamente, que los datos se reciban en el destino configurado y que la respuesta de confirmación se envíe adecuadamente.

### Paso 12: Prueba el flujo completo

Ahora tu formulario de flujo de WhatsApp está listo para funcionar. Cuando tus clientes te envíen un mensaje a tu cuenta de WhatsApp con la palabra clave de activación, tu bot responderá automáticamente y comenzará una conversación presentando el formulario de flujo.

**Escenarios de prueba recomendados:**

1. **Envío exitoso**: Completa el formulario con datos válidos y verifica que la respuesta de confirmación se reciba.
2. **Abandono parcial**: Inicia el formulario pero no lo completes. Verifica que no se envíen datos incompletos.
3. **Datos inválidos**: Ingresa un email sin @ o un teléfono con formato incorrecto. Verifica que se muestren mensajes de error apropiados.
4. **Múltiples envíos**: Envía el formulario varias veces desde el mismo contacto y verifica que los datos se registren correctamente cada vez.

### Paso 13: Monitorea las analíticas

Revisa los datos de los clientes desde la sección de analíticas de E-SMART360. Puedes:

- **Ver datos del informe** (Report Data): Visualiza las analíticas y los datos recopilados del formulario en tiempo real.
- **Descargar datos** (Download Data): Descarga todos los datos recopilados del formulario en formato CSV o Excel desde la página de WhatsApp Flow.


> **Métrica clave:** Monitorea la tasa de finalización del formulario (form completion rate). Si es menor al 60%, considera simplificar el formulario reduciendo la cantidad de campos obligatorios o mejorando la claridad de las instrucciones.

## Elementos del formulario y sus funciones en detalle

Al construir un formulario de flujo de WhatsApp, dispones de varios elementos que puedes arrastrar y soltar. Aquí tienes una guía detallada de cada uno:

### Encabezado (Heading)

Arrastra el elemento **Encabezado** al panel vacío para añadir un título a tu formulario. Para editarlo, haz doble clic en el elemento o usa las opciones que aparecen al pasar el cursor sobre él:

- **Eliminar**: Remueve el elemento del formulario.
- **Editar**: Abre las opciones de configuración del encabezado.
- **Copiar**: Duplica el elemento en el formulario.
- **Mover arriba / Mover abajo**: Reordena la posición del elemento dentro del formulario.

El campo de etiqueta (label) establece el título del formulario visible para los usuarios. Puedes usar formato básico para dar énfasis al texto.

### Campo de texto (Text Field)

Permite a los usuarios ingresar respuestas de texto cortas. Opciones de configuración:

- **Obligatorio**: Marca esta opción para que el campo sea requerido antes de enviar el formulario.
- **Texto de ayuda**: Muestra un texto dentro del campo como guía o placeholder para el usuario.
- **Validación**: Configura validaciones específicas como formato de email, solo números, o longitud mínima y máxima.
- **Campo personalizado**: Asocia el valor ingresado a un campo personalizado del suscriptor.

**Usos comunes**: Nombre completo, correo electrónico, número de teléfono, dirección breve.

### Área de texto (Text Area)

Se utiliza para entradas de texto más grandes, como comentarios, descripciones o feedback detallado. Se expande automáticamente para acomodar respuestas más largas. Opciones de configuración:

- **Altura mínima**: Define cuántas líneas visibles tendrá el campo inicialmente.
- **Límite de caracteres**: Configura un máximo de caracteres permitidos para la respuesta.
- **Obligatorio**: Marca para hacer el campo requerido si es necesario.

**Usos comunes**: Comentarios sobre el servicio, descripciones de problemas técnicos, sugerencias de mejora.

### Grupo de casillas de verificación (Checkbox Group)

Permite a los usuarios seleccionar múltiples opciones de una lista. Ideal para encuestas o selección de intereses múltiples. Configuración disponible:

- **Opciones**: Añade tantas opciones como necesites para que el usuario elija.
- **Selección mínima/máxima**: Define cuántas opciones debe seleccionar el usuario como mínimo o máximo.
- **Disposición**: Elige si las opciones se muestran en formato vertical u horizontal.

**Usos comunes**: Selección de intereses, preferencias de contacto, suscripción a newsletters o categorías de productos.

### Grupo de botones de opción (Radio Group)

Permite a los usuarios seleccionar una sola opción de una lista. Ideal para preguntas de opción única. Configuración similar al grupo de casillas pero limitada a una selección exclusiva.

**Usos comunes**: Método de pago preferido, nivel de satisfacción (1 a 5), tipo de consulta o servicio requerido.

### Campo de fecha (Date Field)

Recopila entradas relacionadas con fechas mediante un calendario interactivo. El usuario puede seleccionar una fecha sin riesgo de formato incorrecto. Configuración:

- **Fecha mínima**: Define la fecha más temprana permitida para evitar selecciones pasadas.
- **Fecha máxima**: Define la fecha más tardía permitida para limitar el rango de selección.
- **Formato**: Elige el formato de visualización (DD/MM/AAAA o MM/DD/AAAA según tu región).

**Usos comunes**: Fecha de nacimiento, fecha de cita médica, fecha de entrega deseada.

### Botón de envío (Submit Button)

Recopila toda la entrada del usuario y la guarda en la base de datos. Es el elemento final que completa el proceso de recolección. Configuración:

- **Texto del botón**: Personaliza el texto visible (ej. "Enviar", "Completar registro", "Finalizar solicitud").
- **Acción post-envío**: Define qué ocurre después del envío (mostrar mensaje de confirmación, redirigir, activar webhook).

## Integración de API HTTP externa en el flujo del bot

La integración de API HTTP de E-SMART360 permite una conectividad perfecta con plataformas externas como WooCommerce, WordPress, Shopify y cualquier sistema con API REST. Esto permite interacciones dinámicas, como mostrar productos de WooCommerce en WhatsApp o integrar publicaciones de WordPress con chatbots.


> **¿Por qué es importante la integración HTTP API?** Permite que los datos recopilados en los formularios de flujo se envíen en tiempo real a sistemas externos como tu CRM, plataforma de email marketing o sistema de tickets, eliminando la necesidad de transferencia manual de datos entre sistemas.

### Cómo configurar una integración de API HTTP paso a paso

#### 1. Accede a la sección de API HTTP

Navega a **Integración > API HTTP** en el panel principal de E-SMART360. Haz clic en el botón **Crear** para iniciar una nueva conexión API.

#### 2. Añade los detalles de conexión

- **Nombre de API**: Proporciona un nombre reconocible como "Crear Usuario WordPress" o "Sincronizar Lead a CRM".
- **Método**: Selecciona el método HTTP apropiado:
  - **GET**: Para obtener datos de un sistema externo.
  - **POST**: Para enviar y crear nuevos datos en el sistema externo.
  - **PUT**: Para actualizar datos existentes.
  - **DELETE**: Para eliminar datos del sistema externo.
- **URL del endpoint**: Ingresa la URL completa de la API externa, por ejemplo: `https://tudominio.com/api/v1/usuarios/crear`.
- **ID de suscriptor de prueba**: Copia un ID de suscriptor del Administrador de Suscriptores. Esto ayuda con las pruebas de API y el paso dinámico de datos durante las pruebas.

#### 3. Configura los encabezados de la solicitud

Los encabezados HTTP son esenciales para que la API externa entienda y procese correctamente tu solicitud:

- **Content-Type**: Normalmente `application/json` para APIs REST modernas. Otras opciones incluyen `application/xml` o `multipart/form-data`.
- **Authorization**: Añade las credenciales necesarias como Bearer Token, API Key o autenticación Basic Auth.
- **Encabezados personalizados**: Puedes añadir encabezados adicionales según lo requiera la API destino, como identificadores de aplicación o versiones de API específicas.

#### 4. Configura el cuerpo de la solicitud

Añade los campos requeridos para la solicitud API. Puedes elegir entre dos tipos de valores:

- **Valores estáticos**: Valores fijos que siempre se envían igual, como un ID de aplicación o un tipo de solicitud predeterminado.
- **Valores dinámicos**: Datos que provienen de la entrada del usuario en el formulario o de campos del suscriptor, como nombre, email o teléfono.

Selecciona el formato del cuerpo según los requisitos de la API destino:
- **JSON**: Formato estándar para APIs REST modernas.
- **Form Data**: Para formularios web tradicionales.
- **X-WWW-FORM-URLENCODED**: Para datos codificados en URL compatibles con formularios HTML.

#### 5. Guarda y verifica la conexión

Haz clic en **Verificar Conexión** para enviar una solicitud de prueba al endpoint configurado. Si la conexión es exitosa, verás una respuesta de confirmación con los datos devueltos por la API externa.


### Solución de problemas de conexión

Si la verificación falla, revisa los siguientes puntos:
- La URL del endpoint debe ser accesible desde internet (no localhost)
- Las credenciales de autenticación deben ser correctas y estar vigentes
- El formato del cuerpo de la solicitud debe coincidir con lo que espera la API
- Los encabezados HTTP deben incluir todos los valores requeridos por la API destino

Una vez que todo funcione correctamente, haz clic en **Guardar API** para finalizar la configuración.

#### 6. Mapea los datos de respuesta

Mapea los campos de respuesta de la API a las variables de E-SMART360. Por ejemplo, si la API devuelve un `user_id` o `email`, puedes mapearlo de vuelta al perfil del suscriptor en E-SMART360. También puedes guardar elementos de lista como JSON para usarlos en mensajes interactivos, como listas dinámicas o catálogos de productos.

#### 7. Usa en el constructor de flujos

Añade un elemento **API HTTP** en cualquier parte del flujo del chatbot. Configura cuándo y cómo debe llamarse la API según las interacciones del usuario. Puedes tener múltiples llamadas API en un mismo flujo para diferentes propósitos, permitiendo crear flujos complejos de múltiples pasos.


> **Consejo avanzado:** Combina formularios de flujo con API HTTP para crear experiencias completas. Por ejemplo, un formulario de registro puede crear automáticamente un usuario en WordPress, enviar un email de bienvenida mediante una API de email marketing y registrar la acción en Google Sheets, todo en una sola interacción del cliente sin intervención manual.

## Automatización de conversaciones complejas con flujo de entrada de usuario

Además de los formularios de flujo de WhatsApp, E-SMART360 ofrece la funcionalidad de **Flujo de Entrada de Usuario (User Input Flow)**, ideal para conversaciones más complejas en el webchat que requieren una interacción paso a paso más guiada.

El Flujo de Entrada de Usuario permite a los chatbots recopilar respuestas de los usuarios de manera estructurada. En lugar de enviar respuestas genéricas, tu chatbot puede hacer preguntas específicas, almacenar las respuestas y usar los datos recopilados para personalizar conversaciones futuras.

### Diferencias entre WhatsApp Flow y User Input Flow

| Característica | WhatsApp Flow | User Input Flow |
|---|---|---|
| Canal principal | WhatsApp | Webchat |
| Formato de interacción | Formularios visuales con campos | Conversación secuencial paso a paso |
| Elementos disponibles | Campos, checkboxes, botones, fechas | Preguntas secuenciales con respuestas |
| Integración visual | Interfaz de formulario estructurado | Diálogo conversacional natural |
| Mejor para | Captura rápida y estructurada de datos | Conversaciones guiadas largas y complejas |
| Personalización | Basada en campos y opciones | Basada en lógica condicional |

### Características clave del User Input Flow

- **Entrada de usuario dinámica**: Recopila y almacena información del cliente en tiempo real durante la conversación.
- **Rutas de conversación personalizadas**: Diseña interacciones del chatbot basadas en las respuestas del usuario, con lógica condicional que adapta el flujo según lo que el usuario responda.
- **Recopilación de datos en tiempo real**: Usa las entradas del cliente de inmediato para proporcionar respuestas personalizadas y relevantes.
- **Almacenamiento estructurado**: Los datos se guardan en el perfil del suscriptor para uso futuro en campañas y segmentación.

### Beneficios de usar User Input Flow

- **Ahorra tiempo**: Reduce el trabajo manual para los equipos de soporte al automatizar la recopilación de información repetitiva.
- **Mejora la experiencia del cliente**: Proporciona respuestas más rápidas y precisas, eliminando transferencias entre departamentos.
- **Recopila datos valiosos**: Obtiene información estructurada del cliente para uso futuro en campañas de marketing y personalización de servicios.
