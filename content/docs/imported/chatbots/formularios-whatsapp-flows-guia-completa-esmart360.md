## Cómo crear un formulario WhatsApp Flow en E-SMART360

Crear un formulario WhatsApp Flow en E-SMART360 es un proceso sencillo. Sigue estos pasos detallados:


### Accede al panel de E-SMART360

Inicia sesión en tu cuenta de E-SMART360 y dirígete al panel de control principal. Desde allí podrás gestionar todos tus canales y bots.

### Selecciona tu bot de WhatsApp

Elige la cuenta de negocio de WhatsApp que tienes vinculada a E-SMART360. Asegúrate de que tu número esté verificado y activo en WhatsApp Business API.

### Inicia un nuevo Flow

Haz clic en "Crear" dentro de la sección "WhatsApp Flow" para abrir el constructor de formularios. Se abrirá el editor visual con una interfaz de arrastrar y soltar.

### Configuración detallada de los campos del formulario

#### Nombre del Flow

Debes asignar un nombre único a tu flujo. Cada vez que crees un nuevo formulario Flow, asegúrate de que el nombre sea distinto a los existentes para evitar conflictos. Un buen nombre descriptivo te ayudará a identificar el propósito del formulario rápidamente.


> Los nombres duplicados causan errores de publicación. Si recibes un error de nombre repetido, simplemente cámbialo por uno nuevo y único antes de continuar.

#### Categorías del Flow

Selecciona el tipo de formulario que deseas crear desde esta categoría. Al hacer clic en esta opción, aparecerá un menú desplegable con múltiples opciones como:

- **Registro de clientes**: Para capturar nuevos leads
- **Soporte técnico**: Para recibir solicitudes de ayuda
- **Pedidos**: Para gestionar compras
- **Reservas**: Para agendar citas
- **Feedback**: Para recolectar opiniones
- **Encuestas**: Para investigación de mercado

#### Nombre único de pantalla

Aquí debes proporcionar un nombre único **sin espacios**. Ten cuidado al escribir, porque mantener espacios entre palabras causará errores. Usa guiones bajos o camelCase si necesitas separar palabras.

Ejemplos correctos: `registro_cliente`, `soporteTecnico`, `formularioPedido2025`

Ejemplos incorrectos: `registro cliente`, `soporte tecnico`

#### Título del formulario

Asigna un título visible a tu formulario WhatsApp Flow. Este título será lo primero que vean tus clientes cuando abran el formulario. Debe ser claro y atractivo.

#### Seleccionar HTTP API

Esta opción te permite seleccionar una conexión HTTP API. Al hacer clic, se abrirá un menú desplegable con todas las conexiones HTTP API que ya hayas creado. Selecciona una y continúa.



### ¿Qué es una HTTP API?

Una HTTP API (Interfaz de Programación de Aplicaciones sobre HTTP) permite la comunicación entre el formulario y sistemas externos. Puedes enviar los datos recolectados a tu CRM, sistema de facturación, Google Sheets, o cualquier otro servicio que acepte peticiones HTTP. En E-SMART360 puedes configurar múltiples conexiones HTTP API y reutilizarlas en diferentes formularios.

### Ejemplo de configuración API

```json
{
  "endpoint": "https://tucrm.com/api/clientes",
  "method": "POST",
  "headers": {
    "Authorization": "Bearer tu_token_api",
    "Content-Type": "application/json"
  },
  "body": {
    "nombre": "{{nombre}}",
    "email": "{{email}}",
    "telefono": "{{telefono}}",
    "mensaje": "{{comentario}}"
  }
}
```

#### Respuesta después del envío

Esta opción te permite configurar una respuesta automática que se enviará después de que tu cliente envíe el formulario. Puedes usar esta respuesta para:

- **Agradecer**: Un mensaje de agradecimiento por completar el formulario
- **Confirmar**: Número de ticket o folio de seguimiento
- **Informar**: Próximos pasos o tiempos de respuesta esperados
- **Ofertar**: Un descuento o promoción especial como incentivo


> Personaliza el mensaje de confirmación incluyendo variables como el nombre del cliente. Por ejemplo: "Gracias {{nombre}}, hemos recibido tu solicitud. Te contactaremos en las próximas 24 horas." Esto hace que la experiencia sea más personal y profesional.

#### Guardar datos en campo personalizado

Aunque puedes consultar los datos de los clientes desde las analíticas de WhatsApp Flow, también puedes elegir un campo personalizado para cada elemento del formulario. Después de arrastrar un elemento y abrirlo, verás un menú desplegable llamado "Campo Personalizado" al final de la configuración.


> Los campos personalizados te permiten organizar y etiquetar los datos recolectados para su uso posterior en campañas de marketing, segmentación de audiencia o informes personalizados. Puedes crear tantos campos personalizados como necesites desde el Gestor de Suscriptores.

### Elementos del formulario y sus funciones

El editor de E-SMART360 incluye varios tipos de elementos que puedes arrastrar al lienzo para construir tu formulario:

**Encabezado (Heading)**
Arrastra el elemento "Encabezado" al tablero vacío para agregar un título a tu formulario. Para editarlo, haz doble clic en el elemento o usa las opciones que aparecen al pasar el cursor sobre él (Eliminar, Editar, Copiar, Mover Arriba, Mover Abajo). El campo de etiqueta define el título del formulario visible para los usuarios.

**Campo de texto**
Permite a los usuarios ingresar respuestas de texto. Puede marcarse como "Obligatorio" en las opciones de edición. Ideal para nombres, correos electrónicos, números de teléfono y cualquier entrada corta.

**Área de texto**
Se utiliza para entradas de texto más grandes, como comentarios o descripciones. Se expande para acomodar respuestas más largas. Perfecto para reseñas, descripciones de problemas o mensajes detallados.

**Grupo de casillas de verificación (Checkbox)**
Permite a los usuarios seleccionar múltiples opciones de una lista. Útil para preferencias múltiples, selección de intereses o confirmación de términos y condiciones.

**Grupo de botones de opción (Radio)**
Permite a los usuarios seleccionar solo una opción de una lista. Ideal para preguntas de opción única como "¿Cómo nos conociste?" o selección de método de pago.

**Campo de fecha**
Se usa para recolectar entradas relacionadas con fechas. Perfecto para fechas de nacimiento, fechas de cita o fechas de entrega preferidas.

**Botón de envío**
Recolecta toda la entrada del usuario y la guarda en la base de datos. Debe ser el último elemento de tu formulario.

```javascript
// Ejemplo de webhook para recibir datos del formulario
app.post('/webhook/whatsapp-flow', (req, res) => {
```

5. Guarda la configuración

## Mejores prácticas para formularios WhatsApp Flow

Optimiza tus formularios siguiendo estas recomendaciones respaldadas por la experiencia de cientos de empresas:


> **Mantén la simplicidad:** Los formularios con más de 7-8 campos tienen tasas de abandono significativamente más altas. Pregunta solo lo esencial en el primer formulario y usa formularios de seguimiento para información adicional.

1. **Comienza con campos obligatorios mínimos**: Solo solicita la información estrictamente necesaria en el primer contacto. El resto puede recolectarse progresivamente.

2. **Usa etiquetas claras y cortas**: Cada campo debe tener una etiqueta que explique exactamente qué información se necesita. Evita ambigüedades.

3. **Aprovecha los campos condicionales**: Muestra u oculta campos según las respuestas anteriores para mantener el formulario relevante y reducir la fatiga del usuario.

4. **Prueba antes de publicar**: Siempre usa la vista previa y prueba el formulario en un dispositivo real antes de ponerlo en producción.

5. **Monitorea las analíticas**: Revisa periódicamente las tasas de finalización y los puntos de abandono para identificar oportunidades de mejora.

6. **Personaliza las respuestas**: Usa variables en los mensajes de confirmación para que cada cliente reciba una respuesta única y personalizada.

```
Número de seguimiento: #{{ticket_id}}

```

## Consideraciones técnicas importantes

### Límites y restricciones



### Campos por formulario

Máximo recomendado: 10-12 campos por formulario para mantener una experiencia de usuario óptima. WhatsApp recomienda no exceder esta cantidad para evitar abandonos.

### Longitud de nombres

Los nombres únicos de pantalla no deben exceder 50 caracteres y no pueden contener espacios. Usa guiones bajos o camelCase.

### Formularios activos

Puedes tener múltiples formularios activos simultáneamente. Cada uno debe tener un nombre de Flow único.

### Solución de problemas comunes

| Problema | Causa | Solución |
|----------|-------|----------|
| Error al publicar | Nombre de Flow duplicado | Cambia el nombre a uno único |
| Formulario no se muestra | Error de sintaxis en nombre de pantalla | Verifica que no tenga espacios |
| Datos no se guardan | Campo personalizado no configurado | Asigna campos personalizados a cada elemento |
| Webhook no funciona | URL incorrecta o sin respuesta | Verifica que el endpoint sea accesible públicamente y devuelva 200 OK |
| API no responde | Token expirado o inválido | Renueva tu token de API y actualiza la conexión |


### ¿Por qué mi formulario no aparece en WhatsApp después de publicarlo?

Esto puede ocurrir por varias razones:
1. El nombre único de pantalla contiene espacios — revísalo y quita cualquier espacio
2. El Flow no está vinculado correctamente al chatbot — verifica la conexión en el flujo del bot
3. La palabra clave configurada no coincide con lo que escribe el cliente — revisa las condiciones de activación
4. El formulario está en estado "Borrador" y no "Publicado" — confirma que hayas hecho clic en Publicar

Si después de verificar todo esto el problema persiste, contacta al soporte técnico de E-SMART360 para asistencia personalizada.

### ¿Qué diferencia hay entre un formulario WhatsApp Flow y un formulario web tradicional?

La principal diferencia es el contexto de uso:
1. **WhatsApp Flow**: El formulario se abre dentro del chat de WhatsApp. El usuario no necesita salir de la aplicación, lo que reduce la fricción y el abandono. La tasa de finalización es significativamente mayor.
2. **Formulario web**: Requiere que el usuario haga clic en un enlace, espere que cargue una página web y luego complete el formulario en un navegador. Esto introduce múltiples puntos de abandono.

Además, WhatsApp Flow se integra nativamente con chatbots y automatizaciones, permitiendo respuestas inmediatas y seguimiento automático.

### ¿Qué sucede si alcanzo el límite de mensajes de WhatsApp Business API?

WhatsApp Business API tiene límites de mensajes basados en el nivel de calidad de tu número. Si alcanzas el límite:
1. Los mensajes salientes se pondrán en cola hasta que el límite se restablezca
2. Los formularios Flow seguirán funcionando para mensajes entrantes (los clientes pueden iniciar conversaciones)
3. E-SMART360 te notificará cuando te acerques al límite
4. Puedes solicitar un aumento de límite desde el Administrador de Negocios de Meta

Para maximizar tu límite, mantén una alta calidad de conversación y evita ser reportado como spam.

### ¿Puedo usar imágenes o videos dentro de un formulario WhatsApp Flow?

### ¿Cómo puedo duplicar un formulario para crear variaciones?

5. Renombra la copia y haz las modificaciones necesarias
6. Publica la nueva versión

Esto te permite crear variaciones para pruebas A/B rápidamente sin tener que reconstruir el formulario desde cero.
